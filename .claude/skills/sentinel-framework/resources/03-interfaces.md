# Interfaces (Ports)

Interfaces define the contracts (ports) that implementations (adapters) must satisfy.
They are generated by Sentinel and must **never** be modified manually.

## Key Concepts

- **Port** = An interface declaring a capability (e.g., `TaskRepository`, `PaymentGateway`)
- **Stereotype** = Categorization: `InboundPort` (driven by user), `OutboundPort` (driven by system)
- **Sealed** = When `sealed: true`, only implementations declared in `sentinel.yaml` can implement it
- **Method signatures** use the format: `methodName(Type param, ...): ReturnType`

## Sealed Interface Rules

When an interface is `sealed: true`:
- **Same-module implementations:** Java `sealed interface ... permits Impl1, Impl2` is generated
- **Cross-module implementations:** ArchUnit rule enforces that only declared implementations exist
- The AI agent **cannot** create new implementations not listed in `sentinel.yaml`
- Implementation classes must use `final class` or `non-sealed class`

## Method Signature Format

```
signature: "methodName(ParamType paramName, ParamType2 paramName2): ReturnType"
```

Examples:
- `save(Task t): void` â€” returns nothing
- `findById(UUID id): Optional<Task>` â€” generic return type
- `process(Payment p): PaymentResult` â€” custom return type
